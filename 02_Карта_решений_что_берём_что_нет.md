# Карта решений: что берём, что нет

Дата: 2026-04-13
Статус: decision table после `01_Большое_исследование_memory_инструментов_2026.md`

## Коротко

Главная стратегия:

> **делаем сами простые организационные слои, берём готовые сложные runtime/search/graph/backend-инструменты только через адаптеры.**

| Слой / задача | Выбор | Статус | Свой или готовый | Почему |
|---|---|---|---|---|
| Фундамент vault | Obsidian + markdown + WikiLinks/MOCs + Properties/Bases | Брать сейчас | делаем сами поверх built-in Obsidian | память читается человеком, переносится и не умирает без runtime |
| Hot memory | SESSION-STATE.md | Брать сейчас | делаем сами | самый дешёвый способ не терять текущую задачу |
| Решения и ошибки | ADR + Known_Bad | Брать сейчас | делаем сами | фиксирует почему выбрали и что не повторять |
| Поиск | QMD + hybrid search | Брать сейчас | готовый QMD + свой adapter/policy | QMD уже близок к vault; hybrid лучше одного поиска |
| Embeddings | local embeddings default; Qwen/Gemini/OpenAI as swappable options | Брать осторожно | адаптер, не hardcode | модель можно менять без миграции памяти |
| Continuity | LCM/Lossless Claw / ClawMem pilot | Брать позже/осторожно | пилот на копии | сначала prove value на длинных задачах |
| Markdown memory tool | Basic Memory / OpenMemory / ClawMem | Проверить | пилот через adapter | похоже полезно, но не отдаём source of truth |
| Wiki compiler | Karpathy pattern + SwarmVault/llm-wiki-kit as references | Брать идею | делаем свой процесс | форма важнее конкретного GitHub |
| Telegram intake | свой bridge + RSSidian-like patterns | Брать сейчас по архитектуре, реализация позже | делаем сами | права, prompt injection, публикации — слишком чувствительно |
| Telegram Adminsha | separate sister agent with limited memory slice | Брать позже после intake | делаем сами/оркестратор | Админша не должна видеть весь vault |
| Comfy memory | Comfy run cards + artifact lineage | Брать позже | делаем свои markdown cards | важен домен, но сначала нужен каркас памяти |
| Visual taste | gold folder + visual scoring cards | Брать позже | свои карточки + later multimodal embeddings | вкус пользователя должен быть объяснимым и проверяемым |
| Echo core | Echo as high-trust local agent; Letta/Mem0 later as experiments | Брать поэтапно | сначала rights+memory slices | личная память слишком чувствительна для раннего внешнего backend |
| Coding memory | ByteRover/Hindsight/Mengram candidates | Брать позже | пилот для вайбкодинга | может ускорить coding agents, но не заменяет project canon |
| Graph | Graphify/Graphiti/Cognee/GraphRAG | Отложить | derived sidecar only | граф полезен после чистого canon + retrieval |
| OpenViking | context filesystem / context DB | Отложить/осторожно | пилот только если нужен dispatcher | мощно, но heavy и может стать новой системой вместо vault |
| Cloud persistence | Fastio/cloud storage | Не брать как core | не использовать для Эхо core | против local-first закрытого контура |
| Aether | не верифицирован | Не брать | проверить отдельно, если найдётся точная ссылка | без ссылки и функции не кладём в roadmap |

## Top-top для Roadmap

В roadmap идут не все инструменты, а только эти решения:

1. **Obsidian/markdown canon**.
2. **SESSION-STATE.md**.
3. **WikiLinks/MOCs/Properties/Bases**.
4. **ADR/Known_Bad**.
5. **QMD + hybrid search**.
6. **Local embeddings first; API embeddings optional**.
7. **Telegram bridge as our adapter**.
8. **Comfy/visual cards as domain memory**.
9. **Echo high-trust core + sister agents with slices**.
10. **LCM/ClawMem/Basic Memory/OpenMemory as pilots, not core**.
11. **Graphify/Graphiti/Cognee only after retrieval and canon**.

## Что не попадает в Roadmap как обязательное

- OpenViking как ранний foundation.
- Mem0/Zep/Letta/LangMem/MemOS/MemMachine как обязательный early backend.
- Cloud-first persistence.
- GraphRAG/Graphiti/Cognee на старте.
- Aether без верификации.

## Простая аналогия

- `markdown vault` — дом;
- `SESSION-STATE` — записка на столе, что делаем прямо сейчас;
- `QMD/hybrid search` — библиотекарь и фонарик;
- `embeddings/vector DB` — смысловой каталог;
- `graph` — карта связей на стене;
- `Mem0/Letta/Zep` — возможные умные помощники-библиотекари;
- `OpenViking` — будущий диспетчер здания, но не фундамент.
